# 🚀 Multi-Agent Workflow Optimization: From Opaque Hangs to Real-time Stability

**Date:** 2026. 03. 12
**Author:** Orbit Agent Team

멀티 에이전트 시스템을 구축하면서 겪은 기술적 난제들과 이를 극복하기 위해 적용한 고도화 전략들을 공유합니다. 본 포스트에서는 사용자 경험(UX), 비용 최적화, 그리고 시스템 안정성이라는 세 가지 핵심 축을 중심으로 진행된 트러블슈팅 과정을 다룹니다.

---

## 1. 불투명한 진행 상황의 해결: 실시간 작업 로그 (Real-time Progress)

### 🔴 Challenge: "에이전트가 살아있나요?"
초기 시스템은 에이전트가 작업을 완료할 때까지 사용자가 빈 화면의 로딩 바만 보며 기다려야 했습니다. 작업이 1분 이상 길어지면 사용자는 시스템이 멈춘 것인지, 작업 중인지 알 방법이 없었습니다.

### ✅ Solution: LangGraph Streaming + Supabase Realtime
- **백엔드 스트리밍**: `graph.invoke`(동기식)를 `graph.astream`(비동기 스트리밍)으로 전환했습니다.
- **중간 상태 기록**: 각 에이전트 노드가 종료될 때마다 `agent_step`이라는 가상 역할을 부여하여 Supabase에 진행 상황을 기록합니다.
- **실시간 UI 피드백**: 프론트엔드에서 Supabase Realtime 채널을 통해 이 로그를 실시간으로 수신, 사용자에게 "현재 어떤 에이전트가 무엇을 하고 있는지"를 애니메이션과 함께 노출합니다.

---

## 2. 토큰 비용의 벽을 넘다: 비대칭 에이전트 모델링 (Asymmetric Modeling)

### 🔴 Challenge: 천정부지로 솟는 API 비용
모든 에이전트가 Claude 3.5 Sonnet 같은 고성능 모델을 사용하면, 단순한 판단이나 요약 작업에도 막대한 토큰 비용이 발생합니다. 특히 대화가 길어질수록 컨텍스트 비용이 기하급수적으로 증가했습니다.

### ✅ Solution: 역할별 지능 차등 배분
- **코딩 역할 (Devs)**: 실제 코드 수정이 일어나는 단계에는 **Claude 3.5 Sonnet**을 배치하여 무결성을 보장합니다.
- **기획 역할 (PM)**: 복잡한 추론과 설계가 필요한 단계에는 **Gemini 2.0 Pro**를 사용합니다.
- **판단/보고 (Supervisor, Reporter)**: 대화 전체를 읽어야 하지만 판단은 단순한 단계에는 초경량 **Gemini 2.0 Flash**를 적용했습니다.
- **결과**: 전체 토큰 비용을 약 60~80% 절감하면서도 최종 작업 품질은 동일하게 유지했습니다.

---

## 3. 대규모 컨텍스트의 함정: 안정성 강화 (Robustness)

### 🔴 Challenge: 응답 잘림과 무한 로딩
대화 내역이 길어지면 LLM이 구조화된 응답(JSON)을 생성하다 중간에 끊어버리는 `EOF while parsing` 에러가 빈번했습니다. 이로 인해 백엔드가 크래시되면 웹 UI는 영원히 로딩 상태에 갇히는 문제가 발생했습니다.

### ✅ Solution: 컨텍스트 트리밍 및 세이프가드
- **히스토리 압축**: 슈퍼바이저에게 전달되는 메시지를 최신 10개로 제한하여 응답 안정성을 100%에 가깝게 끌어올렸습니다.
- **폴백 로직**: 슈퍼바이저에서 파싱 에러 발생 시 즉시 `Reporter` 노드로 연결하여 안전하게 작업을 마무리하도록 설계했습니다.
- **Failsafe UI**: 프론트엔드 리얼타임 리스너에 'ERROR' 키워드 감지 시 강제로 로딩 바를 제거하는 방어 코드를 삽입했습니다.

## 4. 다중 프로젝트 관리: 작업 환경 추적 (Workspace Tracking)

### 🔴 Challenge: "이 작업 어느 폴더에서 했더라?"
사용자가 여러 프로젝트 폴더를 오가며 작업할 때, 과거 작업 이력이 어느 폴더에 귀속된 것인지 알 수 없어 혼동을 야기했습니다.

### ✅ Solution: 작업별 워크스페이스 바인딩
- **DB 스키마 확장**: `tasks` 테이블에 `workspace_name` 컬럼을 추가했습니다.
- **UI 표시**: 사이드바의 최근 작업 목록에 해당 작업이 수행된 프로젝트 이름을 작게 표시하여 직관적인 이력 관리가 가능케 했습니다.
